Java como uma linguagem estrutural

Um programa Java é estruturado de uma maneira específica e particular. Java é uma língua e, portanto, tem gramática semelhante ao Ingles.
gramática das linguagens de programação geralmente é muito, muito mais simples do que as linguas faladas, mas vem com a desvantagem de ter regras mais rígidas.
Aplicar essa estrutura ou gramática à linguagem é o que permite que o computador entenda o programa e o que ele deve fazer.
O programa geral tem uma estrutura , mas também é importante entender o propósito de parte dessa estrutura . Por analogia, um livro didático pode ser dividido em seções, capítulos, parágrafos, frases e palavras (a estrutura), mas também é necessário entender o significado geral das palavras, das frases e dos capítulos para entender completamente o conteúdo do livro. livro didático. Você pode pensar nisso como a semântica do programa.
Uma análise linha por linha do programa deve dar uma ideia melhor tanto da estrutura quanto do significado do programa clássico “Hello World” . Vamos dar uma olhada nisso nas próximas aulas.

Compilando seu primeiro programa em Java!

Para que o computador execute o código Hello World que você escreveu, primeiro ele precisa ser compilado por um compilador Java. O compilador traduz o código de computador escrito em uma linguagem de programação específica para o código de máquina.
O compilador é nada mais que um tradutor que atua como intermediário entre o programador e a CPU do computador.
Os computadores não entendem nativamente as linguagens humanas. Portanto, o compilador está lendo o código escrito pelo programador e traduzindo-o em código de linguagem de máquina que o computador pode executar diretamente.
Em Java os programas não são compilados em arquivos executaveis, eles são compilados em bytecode,que a JVM(Java virtual machine) executa.
O codigo fonte java e compilado em bytecode quando usamos o compilador javac. O bytecode é salvo no disco com a extensão de arquivo .class. Quando o programa deve ser executado, o bytecode é convertido usando o compitlador JIT(Just in time).
O resultado é um codigo de maquina, que e por sua vez alimentado na memoria e executado.
Resumindo, seguimos os dois passos a seguir para executar o programa Java:

  1. Programas Java precisam ser compilados para bytecode .
  2. Então precisamos converter o bytecode em código de máquina .

As classes/bytecode Java são compiladas em código de máquina e carregadas na memória pela JVM quando necessário na primeira vez. Isso é diferente de outras linguagens como C/C++, onde os programas devem ser compilados em código de máquina e vinculados para criar um arquivo executável antes que possa ser executado.

Desafio:Use a declaração 'print'

Imprima no seu console a mensagem "estou aprendendo" com o codigo System.out.println(""); lembre-se de dentro dos parenteses colocar as aspas duplas pois sem ela a frase nao e reconhecida como texto pelo compilador e lembre-se sempre ao final de qualquer linha de codigo colocar o ponto e virgula ";" pois sem ela o compilador não executa o codigo por um erro de sintaxe.
